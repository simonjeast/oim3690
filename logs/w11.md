# Learning Log - Week 11

**Topic:** JavaScript APIs and Asynchronous Programming

## Question 1
- What does `async` do?

## Answer
- `async` declares a function that can use `await` inside it.
- It allows code to wait for things to happen without freezing the page.

## Question 2
- What does `await` do?

## Answer
- `await` pauses a function until a Promise completes (like waiting for API data).
- Example: `const response = await fetch(url);` waits for the response before continuing.
- Only works inside `async` functions.

## Question 3
- What is `fetch()` and how do you use it?

## Answer
- `fetch()` makes HTTP requests to servers or APIs.
- GET: `const response = await fetch(url);`
- POST: `const response = await fetch(url, { method: 'POST', headers: {...}, body: JSON.stringify(data) });`
- Use `await response.json();` to convert the response to a JavaScript object.

## What I learned
- APIs let JavaScript communicate with servers to get or send data over the internet.
- `async`/`await` makes asynchronous code easier to read and write.
- `fetch()` is the modern way to make HTTP requests.
- `headers` provide extra information (like content type or authentication tokens).
- `JSON.stringify()` converts objects to JSON for sending; `.json()` converts responses back.
