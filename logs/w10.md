# Learning Log - Week 10

**Topic:** JavaScript Function Basics

## Question 1
- What's the difference between a function declaration and an arrow function?

## Answer
- A function declaration uses the `function` keyword and has a name, like `function getGrade(score) { ... }`.
- Function declarations are hoisted, so they can be called before they appear in the file.
- An arrow function uses `=>`, like `const getGrade = (score) => { ... }`.
- Arrow functions are not hoisted the same way when stored in a `const`, so they are usually used after they are defined.
- Arrow functions also handle `this` differently, because they keep the `this` value from the surrounding scope.

## Question 2
- When would you use `if...else if` vs a simple `if...else`?

## Answer
- Use a simple `if...else` when there are only two outcomes (true or false).
- Use `if...else if...else` when there are multiple possible conditions to check in order.
- Example: grade logic needs several ranges (A, B, C, D, F), so `if...else if` is the better choice.

## What I learned

- I can return values from functions and use them somewhere else in my program.
- Function declarations and arrow functions can both do similar tasks, but they behave differently with hoisting and `this`.
- Conditionals are easier to read when I choose the right structure for the number of outcomes.
