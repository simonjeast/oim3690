**Topic:** JavaScript Function Basics

## Question 1
- What's the difference between a function declaration and an arrow function?

## Answer
- A function declaration uses the `function` keyword and has a name, like `function getGrade(score) { ... }`.
- Function declarations are hoisted, so they can be called before they appear in the file.
- An arrow function uses `=>`, like `const getGrade = (score) => { ... }`.
- Arrow functions are not hoisted the same way when stored in a `const`, so they are usually used after they are defined.
- Arrow functions also handle `this` differently, because they keep the `this` value from the surrounding scope.

## Question 2
- When would you use `if...else if` vs a simple `if...else`?

## Answer
- Use a simple `if...else` when there are only two outcomes (true or false).
- Use `if...else if...else` when there are multiple possible conditions to check in order.
- Example: grade logic needs several ranges (A, B, C, D, F), so `if...else if` is the better choice.

## Question 3
- What's the difference between push() and unshift()?

## Answer
- `push()` adds a new element to the **end** of an array and returns the new length.
- Example: `const arr = [1, 2, 3]; arr.push(4);` results in `[1, 2, 3, 4]`.
- `unshift()` adds a new element to the **beginning** of an array and returns the new length.
- Example: `const arr = [1, 2, 3]; arr.unshift(0);` results in `[0, 1, 2, 3]`.
- Both methods modify the original array (they're not pure functions).
- `push()` is typically faster because it only adds to the end, while `unshift()` has to shift all existing elements.

## Question 4
- When would you use a for loop vs a while loop?

## Answer
- Use a **for loop** when you know exactly how many times you need to loop (the count is known ahead of time).
  - Example: `for (let i = 0; i < arr.length; i++)` to loop through every item in an array.
  - The for loop handles the counter setup, condition check, and increment automatically.
- Use a **while loop** when you don't know how many iterations you need, but you have a condition to check.
  - Example: `while (input !== "quit")` to keep asking for user input until they type "quit".
  - Example: `while (num > 0)` to count down without knowing the starting number.
- Both do the same thing fundamentally, but a for loop is cleaner when iterating a known amount of times.
- A while loop is better for event-driven or user-input-driven loops where the end isn't predetermined.
