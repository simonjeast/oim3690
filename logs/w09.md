# Learning Log - Week 9

**Topic:** JavaScript Basics

## What I learned
- 'let' is for variables that might change later
- 'const' is for values that should not be reassigned
- JavaScript is case-sensitive, so 'name' and 'Name' are different variables
- Variable names should be clear and usually use camelCase, like 'firstName'
- Main data types we used:
	- Strings: text like '"hello"'
	- Integers: whole numbers like '5'
	- Decimals: numbers like '3.14'
	- Booleans: 'true' or 'false'
	- 'undefined': a variable exists but has no value yet
	- 'null': an intentional empty value
- You can check a value's type with 'typeof', for example 'typeof "hello"' gives '"string"'
- JavaScript does type coercion sometimes, so values can be converted automatically if types are mixed
- '===' checks both value and type, while '==' only checks value after conversion
